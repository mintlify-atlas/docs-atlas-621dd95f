## What is Mi Tienda?

Mi Tienda is a production-ready checkout application built with **React 19** on the frontend and **NestJS** on the backend. It provides a seamless 4-step purchase flow, from product selection through payment confirmation, with secure card tokenization handled directly by Wompi — meaning card data never touches your backend server.

## Key features

<CardGroup cols={2}>
  <Card title="Secure payments" icon="lock">
    Card tokenization happens directly in the browser via Wompi. Raw card numbers never reach the backend.
  </Card>
  <Card title="Real-time status" icon="clock">
    Transaction polling keeps users updated as payments move from `PENDING` to `APPROVED`, `DECLINED`, or `ERROR`.
  </Card>
  <Card title="Clean Architecture" icon="layer-group">
    NestJS backend with strict domain/application/infrastructure layers and a `Result<T, E>` pattern instead of thrown exceptions.
  </Card>
  <Card title="High test coverage" icon="flask">
    125 unit tests across 20 suites with ~99% coverage on all business logic.
  </Card>
  <Card title="Persistent checkout state" icon="floppy-disk">
    Redux Toolkit state is partially persisted to `localStorage`, so users can refresh without losing progress.
  </Card>
  <Card title="Auto-seeded products" icon="box">
    Three sample products are inserted automatically on first boot so the app works without manual setup.
  </Card>
</CardGroup>

## Tech stack

| Layer | Technology |
|-------|-----------|
| Backend | NestJS 11, TypeScript, TypeORM, PostgreSQL 14 |
| Frontend | React 19, Vite 7, Redux Toolkit 2, React Router 7, React Hook Form 7 |
| Payments | Wompi (client-side tokenization, server-side processing) |
| Validation | class-validator (backend), React Hook Form + Luhn algorithm (frontend) |
| API Docs | Swagger / OpenAPI at `/api/docs` |
